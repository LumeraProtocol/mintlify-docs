> ## Eezo-Inlaid Circuitry (v1.17.0)
> 
> This is a feature release, with all accumulated development from the last 3 months. See below for the highlights.
> 
> Note that this release contains multiple critical security fixes, as well as many bug fixes, and is recommended for all users. However, if you are cautious about upgrades, you can also install v1.16.9 which has just the critical security fixes. Specifically, this release fixes [CVE-2026-26313](https://github.com/ethereum/go-ethereum/security/advisories/GHSA-689v-6xwf-5jf3), [CVE-2026-26314](https://github.com/ethereum/go-ethereum/security/advisories/GHSA-2gjw-fg97-vg3r), [CVE-2026-26315](https://github.com/ethereum/go-ethereum/security/advisories/GHSA-m6j8-rg6r-7mv8).
> 
> We recommend recreating your p2p node key after installing this update, which you can do by removing the `DATADIR/geth/nodekey` file before restarting geth. Note this will cause a change in the p2p node ID, which may break static peering setups.
> 
> ## Highlights
> 
> ### Path-based Archive Node with Proofs
> 
> The path-based archive node can now serve proofs (via `eth_getProof`) for the state of older blocks.
> 
> You can configure the block range that supports proving independently from other archive state availability. Specifically, you can use the `--history.trienode` command-line flag to set the amount of blocks for which tree nodes will be tracked.
> 
> This feature is disabled by default. Note that state history cannot easily be recovered once deleted, as it can only be generated by processing blocks. However, you can enable trienode history (and/or state history) at any time to turn a full node into a partial archive node, keeping state from that point in time onwards.
> 
> [#32727](https://redirect.github.com/ethereum/go-ethereum/issues/32727), [#32621](https://redirect.github.com/ethereum/go-ethereum/issues/32621), [#33551](https://redirect.github.com/ethereum/go-ethereum/issues/33551), [#32981](https://redirect.github.com/ethereum/go-ethereum/issues/32981), [#33399](https://redirect.github.com/ethereum/go-ethereum/issues/33399), [#32913](https://redirect.github.com/ethereum/go-ethereum/issues/32913), [#33303](https://redirect.github.com/ethereum/go-ethereum/issues/33303), [#33584](https://redirect.github.com/ethereum/go-ethereum/issues/33584), [#33329](https://redirect.github.com/ethereum/go-ethereum/issues/33329), [#33681](https://redirect.github.com/ethereum/go-ethereum/issues/33681), [#33103](https://redirect.github.com/ethereum/go-ethereum/issues/33103), [#33098](https://redirect.github.com/ethereum/go-ethereum/issues/33098), [#33515](https://redirect.github.com/ethereum/go-ethereum/issues/33515), [#32247](https://redirect.github.com/ethereum/go-ethereum/issues/32247)
> 
> ### EraE History Support
> 
> Geth now suports the [EraE](https://redirect.github.com/eth-clients/e2store-format-specs/pull/16) file format, an archival format for post-merge chain history.
> 
> [#32157](https://redirect.github.com/ethereum/go-ethereum/issues/32157), [#33827](https://redirect.github.com/ethereum/go-ethereum/issues/33827)
> 
> ### OpenTelemetry Tracing
> 
> OpenTelemetry tracing is now supported by the RPC server, including support for distributed tracing. We have also added some tracing spans for block processing via the engine API, i.e. `engine_newPayload`.
> 
> [#33599](https://redirect.github.com/ethereum/go-ethereum/issues/33599), [#33452](https://redirect.github.com/ethereum/go-ethereum/issues/33452), [#33780](https://redirect.github.com/ethereum/go-ethereum/issues/33780), [#33521](https://redirect.github.com/ethereum/go-ethereum/issues/33521)
> 
> ## All Changes
> 
> ### Geth CLI
> 
> - The `geth version-check` subcommand has been removed. This command checked the geth website for signed vulnerability notices, and would tell if updates are necessary ([#33498](https://redirect.github.com/ethereum/go-ethereum/issues/33498))
> - There is now a `--miner.maxblobs` command-line flag to set a limit on blobs included in built blocks ([#33129](https://redirect.github.com/ethereum/go-ethereum/issues/33129), [#33302](https://redirect.github.com/ethereum/go-ethereum/issues/33302))
> - Geth now supports continuous profiling with Grafana Pyroscope ([#33623](https://redirect.github.com/ethereum/go-ethereum/issues/33623))
> - A rare bug that could halt block production in `geth --dev` mode was fixed ([#33146](https://redirect.github.com/ethereum/go-ethereum/issues/33146))
> - A new `--rpc.rangelimit` flag configures the maximum block range for `eth_getLogs` ([#33163](https://redirect.github.com/ethereum/go-ethereum/issues/33163))
> - `geth --exitwhensynced` will now set the `finalized` and `safe` block ([#33038](https://redirect.github.com/ethereum/go-ethereum/issues/33038))
> - `geth --ethstats` now reports the newPayload processing time to the stats server ([#33395](https://redirect.github.com/ethereum/go-ethereum/issues/33395))
> - A lot of minor issues in Geth's command-line flag processing have been fixed ([#33379](https://redirect.github.com/ethereum/go-ethereum/issues/33379), [#33338](https://redirect.github.com/ethereum/go-ethereum/issues/33338), [#33330](https://redirect.github.com/ethereum/go-ethereum/issues/33330), [#32999](https://redirect.github.com/ethereum/go-ethereum/issues/32999), [#33279](https://redirect.github.com/ethereum/go-ethereum/issues/33279), [#33252](https://redirect.github.com/ethereum/go-ethereum/issues/33252))
> - The `evm blocktest` command can now read filenames from stdin when no path is provided ([#32824](https://redirect.github.com/ethereum/go-ethereum/issues/32824))
>
