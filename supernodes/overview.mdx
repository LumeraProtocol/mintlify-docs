A SuperNode is a companion application that runs alongside your Lumera validator node. By operating a SuperNode, you extend your validator's role beyond block production to actively serve the Lumera network's decentralized storage and data-integrity layer — and earn LUME rewards for doing so. Every SuperNode is cryptographically tied to a registered Lumera validator identity, meaning you must already be a validator before you can run one.

## Services

SuperNodes provide two complementary services to the Lumera network:

### Cascade — Permanent, Redundant File Storage

Cascade lets users store files permanently on the Lumera network. When a user submits a Cascade action, the SuperNode encodes the file using **RaptorQ erasure coding**, splits it into redundant chunks, and distributes those chunks across the P2P network. RaptorQ ensures that the original file can be reconstructed even if a subset of chunks is lost, giving Cascade strong durability guarantees without requiring every node to hold a full copy.

### Sense — AI-Powered Duplication Detection

Sense protects the integrity of content registered on Lumera by detecting near-duplicate files before they are stored. When a user submits a Sense action, participating SuperNodes independently evaluate the submission and reach a consensus-backed verdict. This prevents the same content from being registered multiple times under different identities, keeping the network's data layer trustworthy.

## How Reward Earning Works

Lumera distributes LUME rewards to SuperNodes that successfully complete Cascade and Sense actions. The protocol selects a set of **top-ranked SuperNodes** for each action based on stake and performance metrics. If your SuperNode is in the selected set and correctly fulfills the action — storing data, generating RQ identifiers, or returning a Sense verdict — you receive a share of the action fee. Higher stake and reliable uptime improve your ranking and therefore your earning opportunities.

## Network Roles

Two roles interact within the SuperNode layer:

| Role | Description |
|------|-------------|
| **SuperNode Operator** | A registered Lumera validator who runs the SuperNode binary, provides Cascade and Sense services, and earns LUME rewards for fulfilling actions. |
| **User** | Any account that submits a Cascade or Sense action to the Lumera chain, paying an action fee to have the request processed by the top-ranked SuperNodes. |

## Requirements

Before you can run a SuperNode, you must meet the following conditions:

- **Active Lumera validator** — Your node must be registered on-chain as a validator with sufficient stake to appear in the active set. The SuperNode uses your validator's key identity to sign transactions and authenticate with peers.
- **Go installed** — The SuperNode binary is distributed as a pre-built release, but Go is required if you plan to build from source or run tooling.
- **Open network ports** — Port `4444` (gRPC service) and port `4445` (P2P) must be reachable by peers and clients on the network.
- **Lumera node reachable via gRPC** — Your SuperNode connects to your Lumera full node's gRPC endpoint (default `localhost:9090`) to read chain state and submit transactions.

## Architecture

The SuperNode connects to the Lumera chain through its gRPC endpoint to read action requests, verify fees, and submit finalization transactions. It communicates with other SuperNodes over a **peer-to-peer (P2P) network** on port `4445`, using a distributed hash table (DHT) to store and retrieve RaptorQ-encoded data chunks. Clients — such as wallets or applications — interact with the SuperNode's own **gRPC API** on port `4444` to initiate Cascade registrations and downloads.

```
 Client Application
        │
        │ gRPC (port 4444)
        ▼
 ┌─────────────┐        gRPC (port 9090)      ┌──────────────────┐
 │  SuperNode  │ ◄──────────────────────────► │  Lumera Full Node│
 │             │                              │  (your validator)│
 └──────┬──────┘                              └──────────────────┘
        │
        │ P2P DHT (port 4445)
        ▼
 Other SuperNodes
```
