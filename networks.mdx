Lumera Protocol operates two public networks: a production mainnet and an active testnet for development and validator onboarding. Both networks use the same `lumerad` binary—only the chain ID, genesis file, and seed addresses differ. Use the tabs below to find the configuration for the network you want to join.

<Tabs>
  <Tab title="Mainnet">
    ## Mainnet — `lumera-mainnet-1`

    The Lumera mainnet is the production network where real LUME tokens are staked, Cascade files are stored permanently, and Sense results carry economic weight. Treat mainnet keys and configurations with production-grade security practices.

    ### Network Parameters

    | Parameter | Value |
    |---|---|
    | **Chain ID** | `lumera-mainnet-1` |
    | **Native Denom** | `ulume` |
    | **Minimum Gas Price** | `0.025ulume` |
    | **Block Explorer** | Coming soon |

    ### Genesis File

    Download the official mainnet genesis file directly from the Lumera Networks repository:

    ```bash
    curl -o genesis.json https://raw.githubusercontent.com/LumeraProtocol/lumera-networks/refs/heads/master/mainnet/genesis.json
    ```

    Place it in your node's config directory:

    ```bash
    cp genesis.json ~/.lumerad/config/genesis.json
    ```

    ### Seed Nodes

    Add the following seed to `~/.lumerad/config/config.toml` under `[p2p]`:

    ```toml
    seeds = "f64b9efc5747cc3556bdc281dab930a716dd63af@seeds.lumera.io:26656"
    ```

    ### Minimum Gas Price

    Set the minimum gas price in `~/.lumerad/config/app.toml`:

    ```toml
    minimum-gas-prices = "0.025ulume"
    ```

    ### Quick Config Summary

    ```bash
    # Initialize node (replace <moniker> with your node name)
    lumerad init <moniker> --chain-id lumera-mainnet-1

    # Download genesis
    curl -o ~/.lumerad/config/genesis.json \
      https://raw.githubusercontent.com/LumeraProtocol/lumera-networks/refs/heads/master/mainnet/genesis.json

    # Set seeds in config.toml
    sed -i 's/seeds = ""/seeds = "f64b9efc5747cc3556bdc281dab930a716dd63af@seeds.lumera.io:26656"/' \
      ~/.lumerad/config/config.toml

    # Set gas price in app.toml
    sed -i 's/minimum-gas-prices = ""/minimum-gas-prices = "0.025ulume"/' \
      ~/.lumerad/config/app.toml

    # Start the node
    lumerad start
    ```

    <Warning>
      Never share or expose your validator key (`~/.lumerad/config/priv_validator_key.json`) or operator key mnemonic. Back them up securely before starting the node.
    </Warning>
  </Tab>

## Hardware Requirements

Running a Lumera validator or SuperNode requires dedicated infrastructure. The following specifications represent the **minimum recommended hardware** for a production validator. SuperNode operators should provision equivalent or higher specs to handle the additional compute load of Cascade and Sense operations.

| Component | Minimum Specification |
|---|---|
| **CPU** | 8 Cores (modern x86_64 — e.g., AMD EPYC, Intel Xeon) |
| **Memory** | 64 GB RAM |
| **Storage** | 2 TB NVMe SSD (fast random I/O required for P2P and state DB) |
| **Network** | 5 Gbps+ dedicated bandwidth with low latency |
| **OS** | Ubuntu 22.04 LTS or Debian 12 (recommended) |

<Warning>
  Running below these specifications may result in your validator missing blocks, getting jailed, and losing staked LUME through slashing. SuperNode operators running under-spec hardware will fall out of the active top-list and miss rewards.
</Warning>

<Note>
  Storage requirements will grow over time as the chain state and Cascade artifact store expand. Plan for capacity headroom and monitor disk usage regularly.
</Note>
